# 05. 開発ロードマップ

優先度の凡例は [01 要件定義](01-requirements.md) を参照（P0=MVP / P1=初期 / P2=将来）。

## マイルストーン

### M0 — プロジェクト基盤
- Vite + React + TS の scaffold、ESLint/Prettier/Vitest 設定。
- ディレクトリ構成（[04 §3](04-architecture.md)）の骨組み。
- 共有型（[03 §1.4](03-data-model.md)）と定数の置き場を用意。
- CI（lint + test + build）。
- **完了条件**: 空アプリがビルド・デプロイでき、テストが走る。

### M1 — シーンとピン 1 個
- R3F Canvas、カメラ（Orbit）、ライト、グリッド付きベース平面（FR-V1/2/4）。
- 洗濯バサミの 3D アセット（GLB）または暫定プロシージャル形状を表示。
- ローカル座標系・接続点定義（[02](02-clothespin-spec.md)）を `domain/clothespin.ts` に実装。
- **完了条件**: 原点に 1 ピンが正しい寸法・向きで表示される。

### M2 — 連結モデル（コア）
- `domain/solve.ts`（連結→姿勢導出）、`domain/graph.ts`（フォレスト/閉路/占有）。
- Zustand ストア + コマンド（addPin / connectPin / deleteSubtree）。
- 選択 → GRIP ソケット表示 → スナップ連結（FR-P1/2/3）。roll/pitch（FR-P4/5）。
- ドメインの単体テスト（姿勢導出・閉路・占有・整合性）。
- **完了条件**: ベースにルートを置き、ソケット指定でピンを連結・回転・削除できる。

### M3 — MVP 完成
- 色指定・パレット（FR-E6）、Undo/Redo（FR-E5、zundo）。
- 統計（総数・部品表パネル）（FR-S1/2）。
- 保存/読込 JSON（FR-IO1/2、バリデーション + マイグレーション）。
- CSV 部品表出力（FR-IO3）。
- **InstancedMesh** 化と 5,000 ピンのストレステスト（NFR-1）。
- **完了条件**: [01 §8 受け入れ基準](01-requirements.md) を満たす。

### M4 — エクスポート拡充 & 仕上げ
- [x] glTF / STL 出力（FR-IO4/5）、PNG 出力（FR-IO6）。
- [x] プリセット視点・Zoom to fit（FR-V3/5）、全体寸法表示（FR-S3）。
- [x] 複製 ⌘D（FR-E4。クリップボード経由のコピペはバックログ）、切り離し（FR-E3）、配置プレビュー（FR-P8）。
- [x] localStorage 自動バックアップ（FR-IO7）。
- [x] 10,000 ピン描画の確認（NFR-2）、ヘルプ/ショートカット一覧（FR-S4, パネル内）。
- [x] i18n（日英・軽量自前辞書 + パネルの JA/EN 切替。言語設定は localStorage に永続化）。

**M4 完了。** 以降は M5（将来拡張）とバックログ。

## MVP の定義（再掲）

M3 到達 = MVP。「置く・つなぐ・回す・色を付ける・消す・元に戻す・保存/読込・部品表」が
5,000 ピン規模で破綻なく動く状態。

## バックログ（随時整理）

- グリッドスナップの粒度・単位切替（cm/mm 表示）。
- 接続候補のスマート提案（近くのソケット自動ハイライト）。
- ミラー / 配列複製（パターン生成）。
- 測定ツール（2 点間距離）。
- ダーク/ライトテーマ。
