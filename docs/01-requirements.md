# 01. 要件定義書

## 1. 目的・背景

LEGO では [BrickLink Studio](https://www.bricklink.com/v3/studio/main.page) のように、
パーツを 3D 空間で組み立て、設計データを保存・共有・部品表化できるツールが普及している。

本プロジェクト **Clothespin Studio** は、これを「洗濯バサミ（クリップ式）」に置き換え、
**洗濯バサミ同士をクリップで挟み合って大規模な構造物を設計できる Web アプリケーション** を提供する。

設計したモデルは、再編集可能なプロジェクトファイル・部品リスト・3D モデル・画像として
ダウンロードできる。

## 2. スコープ

### 2.1 対象（In Scope）

- 3D 空間での洗濯バサミの配置・連結・編集
- 「クリップで挟み合う」接続点ベースの連結モデルとスナップ
- 色（カラーバリエーション）の指定
- プロジェクトの保存／読込（独自 JSON）
- エクスポート: 部品リスト CSV / 3D モデル glTF・STL / 画像 PNG
- 統計表示（個数・部品表・全体寸法）
- Undo / Redo

### 2.2 対象外（Out of Scope、初期リリース）

- 物理シミュレーション（自重でのたわみ・干渉応力・倒壊判定）
- 複数人同時編集・クラウド保存・アカウント管理（バックエンド非依存で開始）
- 自動組み立て手順書（ステップバイステップ・インストラクション）生成
- 洗濯バサミ以外のパーツ（棒・板・接着剤など）の混在
- 接続のループ（閉路）を含む構造の厳密拘束解決（→ [05](05-roadmap.md) 将来検討）

## 3. 想定ユーザー（ペルソナ）

| ペルソナ | 目的 | 重視する点 |
| --- | --- | --- |
| ホビイスト / 工作好き | 洗濯バサミアートの設計・試作 | 直感的な操作、見た目、共有 |
| 教育者 / ワークショップ主催 | 形状の指導用見本・必要個数の算出 | 部品リスト、画像出力、分かりやすさ |
| メイカー / デザイナー | 大規模構造の検討・3D 出力 | 性能（多数配置）、glTF/STL 出力、寸法精度 |

## 4. 用語定義（Glossary）

| 用語 | 定義 |
| --- | --- |
| ピン (Pin) | 1 個の洗濯バサミ。モデルの最小単位。 |
| JAW コネクタ | ピンが他のピンを挟む側の接続点（オス）。1 ピンに 1 つ。 |
| GRIP ソケット | ピン上の、他のピンの JAW に挟まれうる箇所（メス）。1 ピンに複数。 |
| 接続 (Connection) | あるピンの JAW を、別のピンの GRIP ソケットに嵌めた関係。 |
| ルートピン | どこにも挟まっていない（JAW が未接続の）ピン。構造の起点。 |
| プロジェクト | 1 つの構造物全体（ピン群・色・メタデータ）。保存単位。 |

詳細な座標系・接続点定義は [02 洗濯バサミ仕様](02-clothespin-spec.md) を参照。

## 6. 非機能要件

| ID | 分類 | 要件 |
| --- | --- | --- |
| NFR-1 | 性能 | 5,000 ピン規模で操作可能（中位 PC で 30fps 以上を目標）。インスタンス描画を前提とする。 |
| NFR-2 | 性能 | 10,000 ピン規模まで破綻なく表示できることを努力目標とする。 |
| NFR-3 | 対応環境 | 最新の Chrome / Edge / Firefox / Safari（WebGL2 対応）。デスクトップ優先、タブレット閲覧可。 |
| NFR-4 | オフライン | バックエンド非依存。静的ホスティングで動作（GitHub Pages 等）。将来 PWA 化を検討。 |
| NFR-5 | データ可搬性 | 保存形式はバージョン番号を持ち、後方互換のマイグレーションを可能にする。 |
| NFR-6 | 単位精度 | 内部単位は mm。寸法は実寸（cm）で表示し、誤差は ±0.1mm 未満で丸めない設計とする。 |
| NFR-7 | 国際化 | UI 文言は i18n 化（初期: 日本語 / 英語）。 |
| NFR-8 | アクセシビリティ | 主要操作はキーボードからも可能。コントラスト・フォーカス可視を確保。 |
| NFR-9 | 保守性 | ドメインロジック（連結・拘束解決）を描画から分離し、ユニットテスト可能にする。 |
| NFR-10 | プライバシー | ユーザーデータは端末内で完結。外部送信しない（明示操作を除く）。 |

## 7. 制約・前提

- 全ピンは同一形状（標準サイズ 6×3.9×1.2cm）とする。サイズ違いは将来拡張。
- 連結構造は **フォレスト（森）構造**（各ピンの親は最大 1、子は複数）とする。閉路は初期非対応。
- 物理的成立性（重さ・摩擦・実際に挟めるか）の保証はしない。設計支援ツールとして提供する。

## 8. 受け入れ基準（MVP）

MVP は次が満たされた状態とする（[05 ロードマップ](05-roadmap.md) M3 相当）:

1. ベースにルートピンを置き、選択 → GRIP ソケット指定で新規ピンを連結できる。
2. roll 回転・色指定・削除・Undo/Redo ができる。
3. 5,000 ピンで操作が破綻しない。
4. プロジェクトを JSON で保存 / 読込でき、CSV 部品表を出力できる。
