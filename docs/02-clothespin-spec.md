# 02. 洗濯バサミ仕様（形状・座標系・接続点）

本書は、モデルの最小単位である **ピン（洗濯バサミ）** の幾何・座標系・接続点を定義する。
データモデル / ファイル形式は [03](03-data-model.md) を、これらをどう描画/解くかは [04](04-architecture.md) を参照。

> ⚠️ 本書の数値は **基準仕様（暫定）** である。最終的な接続点座標は、3D アセット（メッシュ）確定時に
> 実測・微調整し、本書を「正」として更新する。アプリ実装はここに定義する**シンボル名（g0…gN, JAW）**に依存し、
> 生の座標値をハードコードしないこと。

## 1. 形状

対象は **クリップ式（スプリングクランプ式）洗濯バサミ**。
プラスチック製の 2 本のレバーが中央のピボットで交差し、金属コイルスプリングで閉じる。
先端（鼻先 = JAW）で対象を挟み、後端（脚 / ハンドル）を握って開く。

形状の基準図（側面ビュー、ローカル座標系・接続点つき）:

## 2. 寸法

| 項目 | 値 |
| --- | --- |
| 長さ（鼻先→後端, X） | 60.0 mm（6.0 cm） |
| 高さ（脚の開き / 側面シルエット, Z） | 39.0 mm（3.9 cm） |
| 厚み（紙面奥行き, Y） | 12.0 mm（1.2 cm） |

- バウンディングボックス: `60.0 × 12.0 × 39.0 mm`（X × Y × Z）。
- 内部単位は **mm**。

## 3. ローカル座標系

各ピンは固有のローカル座標系を持つ。

- **原点**: 金属スプリングの中心（ピボット）。回転の自然な中心。
- **+X**: 鼻先（JAW）方向。長さ方向。
- **+Z**: 高さ方向（側面シルエットの上）。
- **+Y**: 厚み方向（右手系 = `Y = Z × X`）。紙面の奥。

姿勢（回転）は単位クォータニオンで表す。ローカル原点・軸は上記に固定する。

## 4. 接続点

「クリップで挟み合う」結合方式に対応するため、各ピンは 2 種類の接続点を持つ。

### 4.1 JAW コネクタ（オス, male）

- 役割: **他のピンを挟む側**。先端の口。
- 個数: **1 ピンにつき 1 つ**。
- ローカル定義: 位置 `jawOrigin`（鼻先付近）と、挟み込み方向を表す
  - `normal`（口が開く / 噛み込む向き）
  - `axis`（挟んだ対象に沿う方向 = 噛み合わせ線）
- ある接続では、この JAW が相手の GRIP ソケットに嵌る。

### 4.2 GRIP ソケット（メス, female）

- 役割: **他のピンの JAW に挟まれうる面**。
- 個数: 1 ピンに複数（`g0 … gN`）。基準図では暫定 6 点。
- 配置箇所:
  - 脚 / ハンドル（後端側）の上下面 … `g0, g1`（上脚）, `g2, g3`(下脚)
  - 鼻先 / JAW 外面の上下 … `g4, g5`
- 各ソケットはローカルに次を持つ:
  - `position`: ソケット中心
  - `normal`: その面の外向き法線（JAW がこの逆向きから噛む）
  - `tangent`: 面に沿う基準方向（roll=0 の基準）

> 暫定の GRIP ソケット数は 6。脚に沿って離散点を増やす（例: 各脚 3〜4 点）かは、
> 実際の組みやすさを見て [05](05-roadmap.md) で調整する。

### 4.3 接続点テーブル（暫定 / シンボル定義）

実数値は最終アセットで確定。実装はこの **シンボル名** を参照する。

| シンボル | 種別 | 位置の目安 | 法線の目安 |
| --- | --- | --- | --- |
| `JAW` | オス | 鼻先（+X 端） | +X 寄り |
| `g0` | メス | 上脚・前 | +Z |
| `g1` | メス | 上脚・後（後端寄り） | +Z |
| `g2` | メス | 下脚・前 | −Z |
| `g3` | メス | 下脚・後（後端寄り） | −Z |
| `g4` | メス | 鼻先外面・上 | +Z |
| `g5` | メス | 鼻先外面・下 | −Z |

## 5. 連結（スナップ）の幾何

ピン B（親）の GRIP ソケット `g_i` に、ピン A（子）の JAW を嵌めると接続が成立する。
このとき子 A のワールド姿勢は、親 B の姿勢から一意に導出される。

```
T_A_world = T_B_world
          · socketLocal(g_i)        // 親ローカルでのソケット姿勢
          · rollFlip(roll, flip)    // 接続パラメータ（後述）
          · inverse(jawLocal)       // 子の JAW をソケットに合わせる
```

- `socketLocal(g_i)`: position / normal / tangent から作る座標フレーム。
- `jawLocal`: 子の JAW のローカルフレーム。
- `rollFlip(roll, flip)`: 噛み込み法線まわりの離散回転と反転（自由度の離散化）。

### 5.1 接続の自由度

- **roll**: JAW がソケット面の法線まわりに回れる回転。離散ステップで量子化する
  （既定 `90°` / 設定で `15°` などに変更可）。
- **flip**: JAW の表裏反転（180° の付け替え）。真偽値。
- 位置はソケットに固定（連続スライドはしない）。

これにより、各接続は `(parentId, gripIndex, roll, flip)` の少数パラメータで完全に表せる
（[03 データモデル](03-data-model.md) のファイル形式に対応）。

### 5.2 占有・干渉

- 1 つの GRIP ソケットには **1 つの JAW** のみ嵌められる（占有管理）。
- 1 つの JAW は **1 つのソケット** のみに嵌る（→ 親は最大 1）。
- 子は複数のソケットを持つので、1 ピンに複数の子を付けられる（分岐）。
- 自己干渉・他ピンとの貫通は、初期は簡易判定（バウンディング近接）に留める（FR-P7）。

## 6. メッシュ / レンダリング用アセット

- 表示用の 3D メッシュは、本ローカル座標系に整合する単一の GLB を用意する
  （色はマテリアル / 頂点で差し替え）。
- 大量配置のため **InstancedMesh** で描画する（[04](04-architecture.md) 参照）。
- メッシュは LOD を 2〜3 段用意する想定（近接: 詳細 / 遠景: 簡略ボックス）。
- 接続点（JAW/GRIP）は **メッシュとは別のメタデータ**（座標 + 法線 + tangent）として保持し、
  選択時のハイライトやスナップ判定に使う。
