# 02. 洗濯バサミ仕様（形状・座標系・接続点）

本書は、モデルの最小単位である **ピン（洗濯バサミ）** の幾何・座標系・接続点を定義する。
データモデル / ファイル形式は [03](03-data-model.md) を、これらをどう描画/解くかは [04](04-architecture.md) を参照。

> ⚠️ 本書の数値は **基準仕様（暫定）** である。最終的な接続点座標は、3D アセット（メッシュ）確定時に
> 実測・微調整し、本書を「正」として更新する。アプリ実装はここに定義する**シンボル名（g0…g6, JAW）**に依存し、
> 生の座標値をハードコードしないこと。

## 1. 形状

対象は **クリップ式（スプリングクランプ式）洗濯バサミ**。
プラスチック製の 2 本のレバーが中央のピボットで交差し、金属コイルスプリングで閉じる。
先端（鼻先 = JAW）で対象を挟み、後端（脚 / ハンドル）を握って開く。

## 2. 寸法

| 項目 | 値 |
| --- | --- |
| 長さ（鼻先→後端, X） | 60.0 mm（6.0 cm） |
| 高さ（脚の開き / 側面シルエット, Z） | 39.0 mm（3.9 cm） |
| 厚み（紙面奥行き, Y） | 12.0 mm（1.2 cm） |

- バウンディングボックス: `60.0 × 12.0 × 39.0 mm`（X × Y × Z）。
- 内部単位は **mm**。
- 要件の表記「幅 3.9 × 奥行 6 × 高さ 1.2 cm」との対応: 幅 = 高さ Z（3.9cm）、奥行 = 長さ X（6.0cm）、高さ = 厚み Y（1.2cm）。
  本ドキュメント群では混乱を避けるため、以後 **長さ / 高さ / 厚み**（X / Z / Y）の呼称に統一する。

## 3. ローカル座標系

各ピンは固有のローカル座標系を持つ。

- **原点**: 金属スプリングの中心（ピボット）。回転の自然な中心。
- **+X**: 鼻先（JAW）方向。長さ方向。
- **+Z**: 高さ方向（側面シルエットの上）。
- **+Y**: 厚み方向（右手系 = `Y = Z × X`）。紙面の奥。

姿勢（回転）は単位クォータニオンで表す。ローカル原点・軸は上記に固定する。

## 4. 接続点

「クリップで挟み合う」結合方式に対応するため、各ピンは 2 種類の接続点を持つ。

### 4.1 JAW コネクタ（オス, male）

- 役割: **他のピンを挟む側**。先端の口。
- 個数: **1 ピンにつき 1 つ**。
- ローカル定義: 位置 `jawOrigin`（鼻先付近）と、挟み込み方向を表す
  - `normal`（口が開く / 噛み込む向き）
  - `tangent`（挟んだ対象に沿う方向 = 噛み合わせ線。GRIP 側の `tangent` と名称を統一）
- ある接続では、この JAW が相手の GRIP ソケットに嵌る。

### 4.2 GRIP ソケット（メス, female）

- 役割: **他のピンの JAW に挟まれうる箇所**（細い突起・金属リング）。
- 個数: 1 ピンに **7 点**（`g0 … g6`）。
- 配置箇所（いずれも「細い突起」または「金属リング」= 実際に挟みやすい箇所）:
  - 2 本のハンドル（脚）の先端 … `g0`（上ハンドル先端）, `g1`（下ハンドル先端）
  - 2 つのジョー（鼻先）の先端 … `g2`（上ジョー先端）, `g3`（下ジョー先端）
  - 金属スプリングリング上の 3 点 … `g4`（上）, `g5`（ハンドル側 / 後）, `g6`（下）
- 各ソケットはローカルに次を持つ:
  - `position`: ソケット中心
  - `normal`: **子の閉じ軸（squeeze）= roll の回転軸**。子 JAW の閉じ軸（±Z）がこの軸に揃う。
    平先端 = 先端面の法線 / リング = **コイル軸（+Y）**
  - `tangent`: `= normal × outward` = **pitch の回転軸**。
    平先端 = ±Y（幅方向）→ 子は親と**同一平面**（先端タブが子の口幅に収まる）
    リング = 接続点での**局所ワイヤ方向** → 子の幅（Y）がワイヤに沿う = 子は親と**直交**し、
    双方の金属スプリングは**ねじれの位置**になる（物理的に正しい噛み方。ユーザーFB確定）
  - `outward`（定義素）: **基準姿勢（roll=pitch=0）で子ピンの本体が伸びる向き**。
    すべて親の X-Z 面内（部材の延長 / 真上 / 真後ろ / 真下）

> GRIP ソケット数は **7**（ユーザー指定の配置に準拠）。金属スプリングリングは連続した円なので、
> 必要なら `g4–g6` 以外にもリング上の離散点を増やせる（パラメータ化）。実際の組みやすさを見て [05](05-roadmap.md) で調整する。

### 4.3 接続点テーブル（暫定 / シンボル定義）

実数値は最終アセットで確定。実装はこの **シンボル名** を参照する。

| シンボル | 種別 | 位置の目安 | 既定姿勢（outward = 子が伸びる向き） |
| --- | --- | --- | --- |
| `JAW` | オス | 鼻先の口（相手を挟む側） | — |
| `g0` | メス | 上ハンドル（脚）の先端 | ハンドルの延長方向（面内） |
| `g1` | メス | 下ハンドル（脚）の先端 | ハンドルの延長方向（面内） |
| `g2` | メス | 上ジョー（鼻先）の先端 | ジョーの延長方向 ≈ +X（面内） |
| `g3` | メス | 下ジョー（鼻先）の先端 | ジョーの延長方向 ≈ +X（面内） |
| `g4` | メス | 金属スプリングリング・上 | 真上 +Z（面内） |
| `g5` | メス | 金属スプリングリング・ハンドル側（後） | 真後ろ −X（面内） |
| `g6` | メス | 金属スプリングリング・下 | 真下 −Z（面内） |

> 既定姿勢の伸びる向きは「部材の自然な延長」（ユーザー提供のスクリーンショットを正とする）。
> 平先端では子は親と**同一平面**、リングでは子の幅がワイヤに沿うため親と**直交**
> （= スプリング同士がねじれの位置）になる。

## 5. 連結（スナップ）の幾何

ピン B（親）の GRIP ソケット `g_i` に、ピン A（子）の JAW を嵌めると接続が成立する。
このとき子 A のワールド姿勢は、親 B の姿勢から一意に導出される。

```
T_A_world = T_B_world
          · socketLocal(g_i)              // 親ローカルでのソケット姿勢
          · R_normal(roll) · R_tangent(pitch)  // 接続パラメータ
          · FLIP                          // R_tangent(180°): JAW の閉じ軸を normal の逆向きへ
          · inverse(jawLocal)             // 子の JAW をソケットに合わせる
```

- `socketLocal(g_i)`: position / normal / tangent から作る座標フレーム（x = tangent / z = normal）。
- `jawLocal`: 子の JAW のローカルフレーム。
- roll = pitch = 0 のとき、子の本体は `outward = tangent × normal` 方向に伸びる（§4.3 既定姿勢）。

### 5.1 接続の自由度（roll / pitch）

取り付け後に残る回転自由度は、ソケットの形状によって 1 または 2 になる。いずれも **30° 刻み**（決め）。

- **roll**: ソケット normal まわりの回転。「既定姿勢を基準に、子の向きを変える」。
  - 平先端（`g0`–`g3`）: 先端面の法線まわり = 面上のプロペラ回転。
  - リング（`g4`/`g6`）: **コイル軸まわり = ピン平面に平行な面内回転**（ユーザーFB反映）。
- **pitch**: ソケット tangent（= **リングのワイヤ**）まわりの首振り。子をワイヤを軸に振り回す。リング系のみ。
- リング接続では子の幅（Y）がワイヤに沿うため、**子は親と直交し、スプリング同士はねじれの位置**になる
  （ユーザーFB反映: 同一平面のスプリングは物理的に成立しない）。平先端接続は同一平面。

| ソケット | 形状 | 自由度 | roll | pitch |
| --- | --- | --- | --- | --- |
| `g0`–`g3` | 平たい細先端（ハンドル/ジョー） | 1（roll） | ±90°（30° 刻み, 7 値） | — |
| `g4`, `g6` | 丸線リング・上 / 下 | 2（roll + pitch） | ±90°（面内, 7 値） | ±90°（7 値） |
| `g5` | 丸線リング・ハンドル側 | 1（**pitch のみ**） | —（固定。面内回転は親ハンドルと干渉） | ±90°（7 値） |

- `g0`–`g3` の roll を ±90° に制限する理由: それを超えると子が親本体と重なるため（ユーザーFB反映）。
  全範囲が 30° グリッドに整合する（端数なし）。
- **符号規約**: 回転軸（normal / tangent）の正方向から見て **反時計回りを正** とする（右ねじ）。
- **適用順序**: roll（normal まわり）→ pitch（tangent まわり）（`R_normal(roll) · R_tangent(pitch)`、§5 の式）。
- 位置はソケットに固定（線上の連続スライドはしない）。
- 旧仕様の `flip`（表裏反転）は廃止し、向きは roll（+ pitch）で表現する。

これにより、各接続は `(parentId, gripIndex, roll, pitch?)` の少数パラメータで完全に表せる
（[03 データモデル](03-data-model.md) のファイル形式に対応）。`pitch` はリング系（`g4`/`g5`/`g6`）のみ有効。

### 5.2 占有・干渉

- 1 つの GRIP ソケットには **1 つの JAW** のみ嵌められる（占有管理）。
- 1 つの JAW は **1 つのソケット** のみに嵌る（→ 親は最大 1）。
- 親側には複数のソケットがあるので、1 ピンに複数の子を付けられる（分岐）。
- 自己干渉・他ピンとの貫通は、初期は簡易判定（バウンディング近接）に留める（FR-P7）。

## 6. メッシュ / レンダリング用アセット

- 表示用の 3D メッシュは、本ローカル座標系に整合する単一の GLB を用意する
  （色はマテリアル / 頂点で差し替え）。
- 大量配置のため **InstancedMesh** で描画する（[04](04-architecture.md) 参照）。
- メッシュは LOD を 2〜3 段用意する想定（近接: 詳細 / 遠景: 簡略ボックス）。
- 接続点（JAW/GRIP）は **メッシュとは別のメタデータ**（座標 + 法線 + tangent）として保持し、
  選択時のハイライトやスナップ判定に使う。
